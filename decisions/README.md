# Architecture Decision Records (ADRs)

> This folder holds your project's **decision history** — one short file per significant choice.
> This public template includes only the **format guide** and an empty template (`0000-adr-template.md`).
> The real ADRs that document how CoveLab Foundation itself was built ship with the paid product.

---

## Why ADRs

When you (or your AI assistant) come back to a decision weeks later, you want to know *why* it was made — not re-litigate it from scratch. An ADR captures the context and the reasoning once, so the decision sticks and drift is harder.

## How to use

1. Copy `0000-adr-template.md` to a new file, e.g. `0001-my-first-decision.md`.
2. Number them in order. Keep each one short (one screen).
3. Set a status: `Proposed` → `Accepted` → (later) `Superseded by 00XX`.
4. Never delete an ADR — if a decision changes, write a new one and mark the old as superseded. The history is the value.

## Naming

`NNNN-short-kebab-case-title.md` — e.g. `0007-lock-mechanic.md`.
